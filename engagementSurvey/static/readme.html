How to use this survey application

Introduction

This survey software was made for HRMP, for the easy distribution of employee engagement surveys. It is designed to be managed by an administrator at HRMP, who has access to all data generated by client employees. This is to be packaged and distributed to clients in an anonymised form.

Basics

I use $ to refer to the root URL of this project. This project makes extensive use of the Django administrator interface at $/admin/engagementSurvey/ . This is the where the administrator has access to companies, questions and employees, and can directly view response data. These can be added, removed, and changed from this interface.

The basic entities used in this project are Company, Employee, Question, Answer, and each can be changed in the Django admin. This is the main place where questions are added, and associated with a company. Questions can be associated with many companies, but we assume that one company asks one set of questions to all employees. Companies and employees are more difficult to create (but easy to delete) from the Django admin. However, password protected forms are available to easily set up new companies with employees. This is discussed below.

Workflow

Setup

Companies can be added with the password protected form at $/engagementSurvey/new/ . This allows the easy creation of a company with a specifable number of employees. Each company must receive a descriptive name (E.g. Example Company) and a two digit company code (e.g. EC).

Each Employee is associated with a unique id (e.g. EC12345), and a link is generated for that employee to follow and complete the survey. $/engagementSurvey/Example%20Company/EC12345/

Questions must be manually added to each company with the Django admin. A list of all all questions ever asked in the survey application is viewable in the company page in the admin, and a set of question is selected by Ctrl/Command+Click. New questions can be also be added with the '+' button. This means that, at the moment, one company sends the same set of questions to all employees, regardless of employee type and level.

Distribution

Employees must be emailed one link each, which gives them access to the survey. This acts like the login for an employee. Each link can be used once, after which the survey will display a message saying that this link has already been used.

Employees then move to a two page survey:
1. Employee details: Department and level.
2. Actual survey: Questions that have been added to the Company in the admin.

Data

Survey response data is saved in individual answers to questions. These Answer objects are associated with a company and employee.
A password protected results page ($/engagementSurvey/CompanyName/results) shows which employees have responded, along with a list of their responses.
